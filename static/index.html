Data Processing & Model Pipeline

1

Data Load & Parsing

Ingests obfuscated UCI Credit Screening dataset (`crx.data`). Replaces all occurrence of missing value symbols (`?`) with `NaN`. Maps standard variables to categorical and numerical containers.

2

Stratified Split

Splits dataset into 70% Training and 30% Testing sets using stratified sampling to maintain the exact class distribution (55% Rejected, 45% Approved) in both subsets, avoiding sampling bias.

3

Parallel Preprocessing Transformers

Numerical Features (Age, Income, Debt, Score, etc.)

Imputation: Replaces missing numbers with the median of that column to mitigate outlier bias.
Scaling: Normalizes features to range [0, 1] using MinMaxScaler so high range variables (e.g. Income) do not dominate optimization.

Categorical Features (Gender, Married, Education, etc.)

Imputation: Imputes missing values with the most frequent value (mode) of the column.
Encoding: Performs OneHotEncoder(handle_unknown='ignore') to map categories into binary vectors.

4

Optimization via GridSearchCV

Runs a 5-fold cross-validation grid search to isolate the best Logistic Regression hyperparameters. Iterates across:

  • Regularization strength inverse C: [0.001, 0.01, 0.1, 1, 10, 100]
  • Penalty types: L1 (Lasso), L2 (Ridge)
  • Optimizers: liblinear, saga, lbfgs
5

Deployment & Inference

Packages the full preprocessing ColumnTransformer and optimal LogisticRegression estimator inside a unified Pipeline object, guaranteeing zero data leakage and instant predictions on incoming API inputs.
